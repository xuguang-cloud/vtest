## CRITICAL (2 项 — 阻塞发布)

### C1: ADB 命令注入漏洞
**维度**: Secure | **文件**: `ADBAdapter.ts:42-45`
**问题**: `inputText()` 中 `text.replace(/ /g, '%s')` 仅替换空格，未对 `;`, `|`, `$()`, `\``, `>` 等 Shell 特殊字符做完整转义。攻击者可注入任意 ADB 命令。
**影响**: 构造恶意 PRD 文档或测试用例，通过 `inputText` 参数执行 `adb shell rm -rf /sdcard/*`
**修复**: 用 `shell-escape` 或白名单过滤，或改用 `adb shell input text` 的 `--es` 参数:
```typescript
// Replace current escape with:
private escapeText(text: string): string {
  return text.replace(/[;|$()`&><'"]/g, '\\$&').replace(/ /g, '%s')
}
```

### C2: CheckpointManager 无迁移锁
**维度**: Stable | **文件**: `CheckpointManager.ts` + `ExplorationOrchestrator.ts:153`
**问题**: `checkpointManager.saveCheckpoint()` 在每次检查点保存时调用 `db('exploration_checkpoints').insert()`。当数据库锁定时（并发场景或磁盘满），插入静默失败被 catch 吞掉（`ExplorationOrchestrator.ts:155-156`），导致检查点永远丢失。
**影响**: 探索长时间运行后崩溃，恢复时找不到任何检查点
**修复**:
1. `CheckpointManager.saveCheckpoint` 应返回 `Promise<boolean>` 表示是否成功
2. `ExplorationOrchestrator.saveCheckpoint` 应在连续 3 次失败时终止探索
3. 数据库写入增加 WAL 模式 + busy_timeout

## WARNING (8 项 — 发布前重要)

### W1: ExplorationOrchestrator 硬编码 Activity 名
**维度**: Stable | **文件**: `ExplorationOrchestrator.ts:69`
**问题**: `const initialActivity = 'MainActivity'` 硬编码。App 的实际主 Activity 可能不叫 MainActivity。
**修复**: 从 `adb shell dumpsys package <pkg>` 或 `cmd package resolve-activity` 获取真实主 Activity。

### W2: ADB 进程泄漏风险
**维度**: Stable | **文件**: `ADBAdapter.ts:20-40`
**问题**: `spawn()` 返回的 `ChildProcess` 在超时后未显式 `.kill()`。若 ADB 进程卡住超过 timeout 参数，spawn 会 reject Promise，但子进程可能仍存活成为僵尸进程。
**修复**: 在 reject 前调用 `proc.kill('SIGKILL')`，并在 `Promise.race` 中加入超时清理。

### W3: DFSExplorer.explorer 字段为 public
**维度**: Secure/Standard | **文件**: `ExplorationOrchestrator.ts:39`
**问题**: `explorer: DFSExplorer` 被声明为 public。外部代码可直接调用 `orchestrator.explorer.reset()` 破坏探索状态。
**修复**: 改为 `private explorer: DFSExplorer`，或通过 getter 暴露只读访问。

### W4: ExplorationOrchestrator 无异常边界
**维度**: Stable | **文件**: `ExplorationOrchestrator.ts:57-124`
**问题**: `run()` 方法无 try-catch。若 `adb.installAPK()` 抛出异常（APK 损坏/设备断开），整个进程崩溃。
**修复**: 在 `run()` 方法顶层包裹 try-catch，返回 `{ error: string }` 而不是 throw。

### W5: APK 路径无验证
**维度**: Secure | **文件**: `ExplorationOrchestrator.ts:63`
**问题**: `this.adb.installAPK(this.config.apkPath)` 直接接受用户输入的路径，无 `.apk` 扩展名检查、路径遍历检查。
**修复**: 验证路径以 `.apk` 结尾且不包含 `../`。

### W6: ADBAdapter import 路径错误但侥幸工作
**维度**: Standard | **文件**: `ADBAdapter.ts:6`
**问题**: `import { IADBAdapter } from './TestExecutor'` 指向不存在的 `core/adb/TestExecutor`。实际文件在 `services/TestExecutor.ts`。Node 解析失败后回退到 tsconfig paths 或已编译的 dist 才能工作。
**修复**: 改为 `import { IADBAdapter } from '../../services/TestExecutor'`。

### W7: DFSExplorer 无 visitedHashes 去重检查
**维度**: Standard | **文件**: `DFSExplorer.ts:119-120`
**问题**: `this.visitedHashes.add(currentHash)` 总是添加哈希，但从不检查哈希是否已存在。如果用户按 back 回到之前页面，dfsStack 会新建 frame 而非复用（因为同页面但 treeHash 相同），导致不必要的深度消耗。
**修复**: 在 push frame 前检查 `this.visitedHashes.has(currentHash)`。

### W8: 重复的迁移文件目录
**维度**: Sustainable | **文件**: `migrations/` 和 `database/migrations/`
**问题**: 存在两套迁移文件：`src/main/migrations/` 和 `src/main/core/database/migrations/`，表名和字段有重叠。Knex 只会执行一个目录的迁移，导致另一套永远不被应用。
**修复**: 删除 `src/main/migrations/`，统一使用 `src/main/core/database/migrations/`。

## SUGGESTION (6 项 — 可后续迭代)

### S1: `as any` 滥用（32 处）
**维度**: Standard
**问题**: 32 处 `as any` 类型断言，其中 28 处在测试文件（可接受），4 处在生产代码：`TestExecutor.ts:115`, `ExplorationOrchestrator.ts:84`, `main/index.ts`。
**修复**: 为 `ExplorationAction` 的 `STOP` 类型添加 `reason?: string` 到类型定义中，消除 `as any`。

### S2: ExplorationOrchestrator 无进度回调
**维度**: Scalable
**问题**: `run()` 方法是阻塞式全流程，无 `onProgress` 回调。UI 无法显示实时探索进度。
**修复**: 添加 `onStep?: (step: number, action: ExplorationAction) => void` 配置项。

### S3: coverageRate 计算粗糙
**维度**: Standard | **文件**: `ExplorationOrchestrator.ts:119`
**问题**: `coverageRate: this.explorer.getVisitedActivities().size > 0 ? 1 : 0` 二值输出，不反映真实覆盖率。
**修复**: 通过 `adb shell dumpsys package` 获取 App 总 Activity 数，计算 `visited / total`。

### S4: 缺少 AVD 启动前连接验证
**维度**: Stable | **文件**: `ADBAdapter.ts`
**问题**: `ADBAdapter` 构造函数不检查设备是否连接。调用 `adb install` 前无 `isDeviceConnected()` 前置检查。
**修复**: 在 `ExplorationOrchestrator.run()` 开头添加 `if (!(await this.adb.isDeviceConnected())) throw new Error('Device not connected')`。

### S5: delay() 硬编码
**维度**: Scalable | **文件**: `ExplorationOrchestrator.ts:66,108`
**问题**: `await this.delay(2000)` 和 `await this.delay(500)` 硬编码。慢设备上 500ms 不够，快设备上浪费。
**修复**: 改为等待 UI 树变化（轮询 `dumpUITree` 直到变化），或通过配置参数控制。

### S6: 缺少端到端集成测试的 fixture
**维度**: Sustainable
**问题**: E2E 测试 `full-pipeline.e2e.test.ts` 完全依赖 Mock ADB。无真实 ADB 的集成测试 fixture。真实设备测试前无法验证 ADB 命令序列是否正确。
**修复**: 创建 `fixtures/adb-commands.json` 记录预期 ADB 命令序列，在 CI 中做 dry-run 验证。
