## 一、产品概述

### 1.1 产品定位

| 字段 | 内容 |
|------|------|
| 产品名称 | VTest |
| 产品形态 | Electron 桌面端（主控）+ Web 报告端（查看/协作）|
| 产品定位 | 开源核心 + 企业版付费（混合模式）|
| 目标用户 | QA工程师、测试开发、移动端开发（Android优先）|
| 核心价值 | AI自动探索App全路径，生成验收级测试用例，减少80%真机回归测试 |
| MVP周期 | 3个月极速验证 |

### 1.2 问题陈述

移动App测试面临三大痛点：
1. **真机依赖重**：设备采购、维护成本高，覆盖机型有限
2. **用例编写慢**：人工遍历App路径耗时，且容易遗漏边界场景
3. **需求符合性难验证**：PRD/设计稿与最终实现的一致性，依赖人工逐页核对

### 1.3 解决方案

VTest 通过「AI自主探索 + 需求符合性比对」双引擎，自动完成：
- 遍历App所有可交互路径
- 将探索结果与PRD需求、设计稿自动比对
- 生成验收级测试用例和执行报告
- 自动分类Bug（需求缺失/设计缺陷/UX缺陷/实现不一致）

## 二、用户故事（MVP核心）

| ID | 角色 | 故事 | 验收标准 |
|---|------|------|---------|
| US-01 | QA工程师 | 我能导入一个APK文件，系统自动安装到模拟器 | 安装成功提示；失败有明确错误日志 |
| US-02 | QA工程师 | 我能启动AI探索，系统自动遍历App所有可点击路径 | 探索完成后生成路径树；覆盖率可量化 |
| US-03 | QA工程师 | 我能查看AI自动生成的测试用例列表 | 每条用例包含：步骤、预期结果、优先级 |
| US-04 | QA工程师 | 我能执行生成的测试用例并查看结果 | 执行结果：通过/失败/阻塞；失败有截图证据 |
| US-05 | QA工程师 | 我能在Web端查看测试报告 | 报告包含：通过率、失败详情、截图对比、路径覆盖率 |
| **US-06** | **QA工程师/PM** | **AI探索时，系统自动将每条路径的实际UI展示、交互行为与PRD需求文档和设计稿进行对比，识别需求缺失、设计缺陷、UX问题，并自动归类写入报告。PRD和设计稿支持从钉钉、飞书、Figma及用户有权限的网页直接获取** | **1. 支持导入结构化PRD（JSON）和设计稿（图片/标注）<br>2. 支持从钉钉文档、飞书文档、Figma、及用户已授权访问的网页URL自动抓取PRD和设计稿<br>3. 探索过程中实时对比：UI元素是否存在、交互行为是否符合预期<br>4. 发现差异自动生成Bug，类型分为：`需求缺失`/`设计缺陷`/`UX缺陷`/`实现不一致`<br>5. Bug包含：截图证据、路径复现步骤、PRD原文引用、严重等级（P0-P3）** |

## 三、Bug分类定义（US-06核心）

| Bug类型 | 定义 | 示例 |
|---------|------|------|
| `需求缺失` | PRD有定义但App未实现 | PRD要求登录页有「忘记密码」，实际没有 |
| `设计缺陷` | UI/交互与设计稿不一致 | 按钮颜色应为#FF6600，实际为#CCCCCC |
| `UX缺陷` | 功能实现但用户体验违反规范 | 弹窗无关闭按钮，用户无法退出 |
| `实现不一致` | 相同功能在不同路径下表现不一致 | 首页搜索框有防抖，详情页搜索框没有 |

## 五、文档源集成策略（US-06 核心支撑）

### 5.1 PRD/设计稿来源矩阵

VTest 的 PRD 和设计稿支持从以下来源获取，统一转换为内部结构化格式后参与比对：

| 来源类型 | 平台 | MVP支持度 | 实现方式 | 数据流向 |
|---------|------|----------|---------|---------|
| **IM协作平台** | 钉钉文档 | ✅ MVP | OAuth2授权 → 调用钉钉开放API → 拉取文档内容/表格 | 钉钉API → VTest结构化JSON |
| **IM协作平台** | 飞书文档 | ✅ MVP | OAuth2授权 → 调用飞书开放API → 拉取文档内容/多维表格 | 飞书API → VTest结构化JSON |
| **设计工具** | Figma | ✅ MVP | Personal Access Token → Figma REST API → 拉取Frame/节点树/标注 | Figma API → VTest结构化JSON |
| **通用网页** | 任意已授权URL | ✅ MVP | 用户在VTest内嵌浏览器登录 → Agent抓取页面DOM/截图 | 浏览器DOM → AI结构化提取 |
| **本地文件** | JSON/Word/PDF/MD | ✅ MVP | 本地文件导入 → 解析器转换 | 文件解析 → VTest结构化JSON |
| **AI深度解析** | 任意自然语言文档 | ⬜ 企业版 | LLM多轮推理 → 自动提取需求条目+UI约束 | LLM → VTest结构化JSON |

### 5.3 网页抓取方案（已授权网页）

| 步骤 | 描述 |
|------|------|
| 1. 用户在VTest内嵌浏览器中打开目标URL | 复用用户已有的登录态/Cookie |
| 2. VTest Agent导航到PRD/设计稿页面 | 自动识别页面类型（表格/文档/图片） |
| 3. DOM抓取 + 截图 | 提取文本结构、表格数据、图片资源 |
| 4. AI结构化提取 | LLM将非结构化内容转换为VTestDoc JSON |
| 5. 缓存 + 版本管理 | 支持增量更新，避免重复抓取 |

### 5.4 PRD格式支持（Option C 实现策略）

| 阶段 | 格式 | 实现方式 | 准确度 |
|------|------|---------|--------|
| **MVP v1.0** | 结构化JSON（本地导入） | 用户在VTest内录入/导入结构化需求条目 + 设计稿截图标注 | 高（可控）|
| **MVP v1.0** | 钉钉/飞书文档 | OAuth2授权 → API拉取 → AI结构化提取 | 中高（API数据结构化程度高）|
| **MVP v1.0** | Figma设计稿 | Token授权 → REST API拉取Frame节点树+标注 | 高（Figma API结构化）|
| **MVP v1.0** | 任意已授权网页 | 内嵌浏览器 → DOM抓取 → AI结构化提取 | 中（依赖页面结构）|
| **企业版 v2.0** | 自然语言文档深度解析 | 多模态LLM端到端解析 → 需求条目+UI约束+交互规则 | 中高（需人工复核）|

        "输入错误密码，提示「账号或密码错误」"

      "uiRequirements": [
        { "element": "login_button", "type": "button", "color": "#FF6600", "text": "登录" },
        { "element": "username_input", "type": "input", "placeholder": "请输入用户名" }
      ],

    }
  ]
}

## 六、非功能性需求

| 类别 | 要求 | 验证方式 |
|------|------|---------|
| 性能 | AI探索：中等复杂度App（<20个Activity）在30分钟内完成全路径探索 | 基准测试 |
| 性能 | 测试执行：用例回放速度不低于人工操作的80% | 基准测试 |
| 兼容性 | Android 8.0 - 14.0；x86_64模拟器（Android Emulator）| 多版本验证 |
| 稳定性 | 单次探索任务崩溃率 < 1% | 监控日志 |
| 稳定性 | 测试执行失败率（非App Bug导致的） < 5% | 监控日志 |
| 安全性 | APK文件沙箱运行（独立AVD实例）| 架构审查 |
| 安全性 | 不访问用户真实数据（通讯录/相册等）| 权限审查 |
| 安全性 | IM/OAuth2 Token加密存储（AES-256），不过度请求权限 | 安全审计 |
| 安全性 | 网页抓取仅限用户主动授权的URL，不爬取未授权页面 | 安全审计 |
| 可扩展性 | 模块化设计，iOS/鸿蒙引擎可插拔替换 | 架构审查 |
| 可维护性 | TDD开发模式，功能/性能/接口测试覆盖率 > 80% | CI自动检查 |

## 九、风险与缓解

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| AI探索路径爆炸（指数级）| 无法在合理时间内完成探索 | DFS + 哈希去重 + 最大深度限制 + Activity去重 |
| 截图对比误报率高 | US-06产生大量误报Bug | 阈值可调 + 人工复核标记 + 模型持续训练 |
| Android模拟器性能差 | 探索/执行速度慢 | x86_64镜像 + 硬件加速（HAXM/Hyper-V）+ 并行多AVD |
| PRD结构化录入成本高 | 用户不愿意手动录入 | 支持钉钉/飞书/Figma/网页直接拉取 + 企业版AI自动解析 + MVP提供JSON模板生成工具 |
| 与真实设备差异 | 模拟器无法完全替代真机 | 明确告知用户：模拟器用于功能验证，真机用于性能/兼容性最终验收 |
| IM授权安全风险 | 用户担心钉钉/飞书Token泄露 | AES-256加密存储 + 最小权限原则 + 支持随时撤销授权 |

> - **Standard（标准）**：需求描述清晰、可测试、无歧义
